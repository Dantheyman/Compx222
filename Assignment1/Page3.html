What is a request?

A HTTP request is made by a client,to a host, which is located on a server. The purpose of a request is to get access to a resource located on the server. The method used for the request then determines the interaction between the client and the resource.

How is a request formatted?

Format for a request.

  • Request line
  • Request header fields
  • An empty line
  • Optional HTTP body

Request line

The Request line consists of at least 3 items(IBM,2021).

A method.
The method is a one word command that instructs the server what the client wants done to the resource. E.g. The client could ask the server to send that particular resource to the client.
The path
The path component for the URL of the server. The path identifies where to find the resource on the server.
The HTTP Version number.
Letting the server know the HTTP specification that the request is complying too.

Additional items a request line might include are.

Query String.
This provides a string that the resource can use for some process. It is put after the Path and is preceded by a question mark.
The Scheme and host components of the URL.
This is also known as the absolute URL. For some versions of HTTP if the Scheme and host components aren’t included in the request then they must be included in a Host header.

Request headers

Headers are written on requests to provide the recipient information from the sender about the message or the way it wants to communicate. This influences the way the recipient will respond to the sender. A Header consists of a name and a value(IBM,2021).

Message body

The Message body contains the entity body which is the actual content of the request. Not all requests have a message body. For example POST method has a message body consisting of the information being sent to the server while a GET request has no data to send so does not have a message body(IBM,2021).

Error Codes

Evey request that is sent to a server is responded with a HTTP status code. Some of these are used to identify errors. According to Anicas(2021) common error codes include

400 - Bad Request
Occurs when the request sent has invalid syntax
401-Unauthorized
Occurs when either the client trying to gain access to the resource hasn’t been authenticated by the server or hasn’t been authenticated correctly.
404 - Not Found
Occurs when even though the client can communicate with the server it is unable to find the requested resource.
502 - Bad Gateway
Occurs when the server is unable to connect to its backend servers that actually fulfill the request.
